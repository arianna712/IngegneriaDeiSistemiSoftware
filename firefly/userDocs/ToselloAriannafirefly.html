Firefly Sprint A

Introduction

Goal:Il progetto firefly ha come scopo quello di simmulare il comportamento di una lucciola.

Requirements

Definire un modello qak del sistema firefly che simula il comportamento di una lucciola.

Requirement analysis

Il sistema firefly ha lo scopo di modellare una lucciola che manifesta un comportamento ciclico di accensione e spegnimento. Il comportamento della lucciola deve essere rappresentato da un QActor, che alterna stati di accensione e spegnimento a intervalli di tempo variabili, determinati da un timer interno. Il sistema deve inoltre prevedere una modalità di visualizzazione del comportamento della lucciola. A tal fine, si utilizza un progetto separato denominato griddisplay, che consente la rappresentazione grafica tramite una pagina HTML contenente una griglia. In tale griglia, un quadratino rappresenta la lucciola e ne visualizza lo stato (acceso/spento). 

	Context ctxfirefly ip [ host="localhost" port=8040 ]
	Context ctxgrid ip [ host="127.0.0.1" port=8050 ]

	ExternalQActor griddisplay context ctxgrid

	QActor firefly1 context ctxfirefly {
	    [#
	        var X = 1
	        var Y = 1
	    #]

	    State s0 initial {
	        printCurrentMessage color green
	    }

	    Goto flash
	}

Problem analysis

Il comportamento della lucciola può essere modellato come un ciclo continuo di due stati: acceso e spento, gestiti da un actor che esegue un loop di esecuzione. L’intervallo temporale tra le transizioni di stato non è fisso, ma viene generato dinamicamente tramite un timer interno, che può assumere valori casuali in un intervallo prefissato (ad esempio tra 500ms e 1000ms). 

State flash{

    [# var Timer = java.util.Random().nextLong(500L,1000L) #]

    delayVar Timer

    // accensione lucciola
    forward griddisplay -m cellstate : cellstate($X, $Y, 1)

    delayVar Timer

    // spegnimento lucciola
    forward griddisplay -m cellstate : cellstate($X, $Y, 0)

    printCurrentMessage color yellow
}

Goto flash
