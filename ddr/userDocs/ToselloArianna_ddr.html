DDR Sprint 1

Introduction

Goal: Costruire un sistema software che comanda un Differential Drive Robot (DDR) in modo che, partendo dalla posizione iniziale HOME, il robot si sposti lungo il perimetro di una stanza rettangolare vuota

Requirements

Costruire un sistema software che comanda un Differential Drive Robot (DDR) in modo che, partendo dalla posizione iniziale HOME, il robot si sposti lungo il perimetro di una stanza rettangolare vuota. Il DDR si deve fermare quando tornato in HOME. Il DDR è anche sensibile ai dati rilevati da un Radar. Quando il Radar rileva un 'intruso' a una distanza minore di una distanza prefissata ``DMIN``:
  • se il DDR si sta muovendo , il DDR si ferma fino a che il Radar non rileva più un 'intruso' così vicino
  • se il DDR è fermo in HOME, il DDR ruota su sè stesso fino a che un 'intruso' così vicino scompare

Requirement analysis

Il sistema software analizzato è composto da diverse entità:
  • HOME: nei requisiti rappresenta sia la posizione iniziale del DDR all'interno della stanza sia la posizione finale del percorso lungo il perimetro.
    Per poter formalizzare questa entità è necessario prima definire in modo preciso il modello della stanza. Essa può essere rappresentata come un rettangolo in un piano euclideo, caratterizzato da una base B e un'altezza H, entrambe espresse in metri. Introducendo un sistema di assi cartesiani, ogni punto della stanza può essere identificato tramite una coppia di coordinate (X,Y). Tuttavia, il robot non può essere considerato come un punto materiale, poiché occupa uno spazio fisico. Si assume quindi che esso abbia una forma quadrata di lato R. Di conseguenza, risulta più opportuno discretizzare l'ambiente, rappresentando la stanza come una griglia composta da celle, ciascuna corrispondente a un'unità robotica compatibile con le dimensioni del robot stesso.
    All'interno di questa rappresentazione, la HOME viene definita come un'area in grado di contenere completamente il robot. In pratica, essa corrisponde a una cella predefinita della griglia che modella la stanza e che rappresenta sia il punto di partenza sia quello di arrivo del percorso.
    Possiamo formalizzare quanto detto con il seguente codice Kotlin: 
    • 
[# 
	/**
 	 * Stanza DDR
	 *
 	 * Sistema di riferimento: asse x verso destra, asse y verso il basso (convenzione grafica).
 	 *
 	 * @property B larghezza della stanza in unità robotiche (numero di celle su x)
 	 * @property H altezza della stanza in unità robotiche (numero di celle su y)
  	 * @property R lato dell'unità robotica in metri — coincide con il lato di ogni cella
 	 *
 	 * Dimensioni fisiche della stanza: (B * R) metri × (H * R) metri
 	 */
	data class Stanza(
    	val B: Double,
    	val H: Double,
    	val R: Double
	) {
	    /** Larghezza fisica della stanza in metri */
	    val larghezzaMetri: Double get() = B * R

    	/** Altezza fisica della stanza in metri */
    	val altezzaMetri: Double get() = H * R

    	/** Posizione HOME: angolo in alto a sinistra, cella (0, 0) */
    	val home: Posizione = Posizione(0, 0)
	}

	/**
 	 * Posizione nella griglia, espressa in unità robotiche (celle).
 	 * (0, 0) corrisponde a HOME (angolo in alto a sinistra).
 	 *
 	 * @property x colonna (asse orizzontale, cresce verso destra)
  	 * @property y riga   (asse verticale,   cresce verso il basso)
	 */
	data class Posizione(val x: Int, val y: Int)
#]
  • Radar: è un'entità proattiva che, a intervalli regolari, monitora l'area circostante il DDR con lo scopo di individuare eventuali “intrusi” presenti a una distanza inferiore a D_MIN La caratteristica principale del radar è quella di fornire informazioni al sistema, in particolare la distanza rilevata D, espressa in metri. Poiché è disponibile il linguaggio custom QAK, si è scelto di modellare il radar come un attore.
    Affinché tali informazioni possano essere effettivamente utilizzate dal sistema per adattare il comportamento del DDR, è necessario definire in modo esplicito il meccanismo attraverso cui esse vengono rese accessibili. In generale, si possono considerare due possibili approcci:
    1. Il radar può inviare le informazioni rilevate tramite un evento, poiché non è a conoscenza a priori del destinatario che dovrà elaborarle. In questo caso, la comunicazione avviene in modalità event-driven, permettendo la diffusione del dato a tutti i componenti interessati del sistema.
    Si formalizza quindi il seguente attore: 
    
		System ddrsystem

		/*
		* MODELLO DELL'ANALISTA
		*/
		
		// -----definizione dei messaggi-----
		Event radar : distance(D)

		// -----definizione del contesto-----
		Context ctxddrsys	ip [ host="localhost" 	port=8010 ]

		// -----definizione degli attori-----
		QActor radar context ctxddrsys {
			[# 
				var D = 0
				var Timer = 1000L
			#]
			State s0 initial {
				println("$name | starting") color green
			}
			Goto rileva_distanza
			
			State rileva_distanza{
				// rilevamento della distanza D (in metri)
				emit radar : distance(D)
			}
			Transition t0 
				whenTimeVar Timer -> rileva_distanza
		}

    2. altrimenti, il radar potrebbe comunicare a distanza rilevata in risposta ad una richiesta del sistema. In questo caso formalizzo il seguente attore: 
    
		System ddrsystem

		/*
		* MODELLO DELL'ANALISTA
		*/
		
		// -----definizione dei messaggi-----
		Request getDistance : getDistance(D) "richiesta del sistema"
		Reply   radar : distance(D)   for getDistance

		// -----definizione del contesto-----
		Context ctxddrsys	ip [ host="localhost" 	port=8010 ]

		// -----definizione degli attori-----
		QActor radar context ctxddrsys {
			[# 
				var D = 0
				var Timer = 1000L
			#]
			State s0 initial {
				println("$name | starting") color green
			}
			Goto rileva_distanza
			
			State rileva_distanza{
				// rilevamento della distanza D (in metri)
			}
			Transition t0 
				whenTimeVar Timer -> rileva_distanza
				whenRequest getDistance -> handleRequest
		
			State handleRequest{
				onMsg( getDistance : getDistance(D) ){
					replyTo getDistance with radar : distance( $D )  
				} 
			}  
			Transition t0 
				whenTimeVar Timer -> rileva_distanza
		}
